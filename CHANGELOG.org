* 0.2.0 — 2026-03-21

** Added
- ISO 8601 default timestamp format (=%Y-%m-%d=, e.g. =2026-03-21=)
- Format-aware date parser: handles both ISO 8601 and legacy =MM/DD/YY= formats
- =or-east-normalize-timestamps= command to batch-rewrite all node timestamps (including zero-sentinel normalization to =0000-00-00=)
- =or-east--zero-sentinel-p= helper for detecting all-zeros dates in any format
- Comprehensive docstrings for all public and internal functions
- =;;; Section= headers organizing =or-east-mode.el= by concern
- Commentary sections in test files
- =;;;###autoload= cookies for =or-east-disable= and =or-east-toggle=
- CHANGELOG.org, CODE_OF_CONDUCT.md, .dir-locals.el
- "How it works" section and usage examples in README
- SPDX license identifier in file header

** Changed
- Default =or-east-node-stat-format-time-string= from ="%D"= to ="%Y-%m-%d"=
- =or-east--parse-date= now uses =parse-time-string= with MM/DD/YY fallback instead of hardcoded regex
- Renamed =or-east-node-handle-modified-time-tracking-h= to =or-east--setup-modified-tracking=
- Replaced =(eq 'string (type-of id))= with =(stringp id)=
- Replaced =(or file-path nil)= with =file-path=

** Removed
- Unused =or-east-link-replace-at-point= function
- Stale TODO comment

* 0.1.0 — 2022-12-28

** Added
- =or-east-mode= minor mode with automatic stat tracking
- =last-accessed=, =last-modified=, =last-linked= property tracking
- Body-change detection via =buffer-hash=
- Activity scoring system with configurable weights and decay
- =or-east-node-sort-by-activity= for =org-roam-node-find= integration
- Buttercup test suite
